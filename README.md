Проект: База данных "Компьютерный клуб"

О проекте

Этот проект представляет собой реляционную базу данных, разработанную для моделирования работы компьютерного клуба. Система отслеживает:

Клиентов и их дисконтные карты

Компьютеры и их текущее состояние

Услуги клуба (еда, напитки, игровое время)

Игровые сессии (аренда компьютеров)

Заказы дополнительных услуг во время сессий

Продажи товаров и услуг

База данных может служить основой для реальной системы учета компьютерного клуба или использоваться в учебных целях для изучения проектирования реляционных моделей и нормализации данных.

Сущности (таблицы)
1. Card (Карты)
Дисконтные карты клиентов для накопления баллов и оплаты.

card_id — ID карты (первичный ключ)

code — Уникальный номер карты

balance — Баланс на карте (по умолчанию 0)

2. Clients (Клиенты)
Зарегистрированные клиенты клуба.

clients_id — ID клиента (первичный ключ)

full_name — ФИО клиента (обязательно)

phone — Номер телефона

birthday — Дата рождения

card_id — Ссылка на карту клиента (может быть NULL)

3. Computers (Компьютеры)
Игровые компьютеры клуба.

computers_id — ID компьютера (первичный ключ)

computer_code — Уникальный код компьютера

status — Статус компьютера: Available (доступен), Busy (занят), Maintenance (на обслуживании)

price_per_hour — Стоимость аренды за час

4. Services (Услуги)
Дополнительные услуги и товары клуба.

services_id — ID услуги (первичный ключ)

name — Название услуги/товара

category — Категория: Food (еда), Drink (напитки), Other (прочее)

price — Цена

5. Sessions (Сессии)
Игровые сессии (аренда компьютеров).

sessions_id — ID сессии (первичный ключ)

client_id — ID клиента

computer_id — ID компьютера

start_time — Время начала сессии

end_time — Время окончания сессии

total_price — Общая стоимость сессии

6. Orders (Заказы)
Заказы дополнительных услуг.

orders_id — ID заказа (первичный ключ)

client_id — ID клиента

session_id — ID сессии (может быть NULL для заказов без аренды)

orders_time — Время заказа (по умолчанию текущее время)

7. Orders_Item (Позиции заказа)
Состав заказов (список услуг/товаров).

orders_item_id — ID позиции заказа (первичный ключ)

orders_id — ID заказа

service_id — ID услуги

quantity — Количество (по умолчанию 1)

unit_price — Цена за единицу

Типы данных (ENUM)
computer_status
Статусы компьютеров:

'Available' — Доступен для аренды

'Busy' — Занят (идет сессия)

'Maintenance' — На обслуживании

service_category
Категории услуг:

'Food' — Еда

'Drink' — Напитки

'Other' — Прочие услуги

Связи между сущностями
Основные связи:
Card ↔ Clients (1 к 1)

Одна карта принадлежит одному клиенту

Клиент может иметь карту (или не иметь)

Clients.card_id → Card.card_id

Clients ↔ Sessions (1 ко многим)

Один клиент может иметь много сессий

Одна сессия принадлежит одному клиенту

Sessions.client_id → Clients.clients_id

Computers ↔ Sessions (1 ко многим)

На одном компьютере может быть много сессий

Одна сессия использует один компьютер

Sessions.computer_id → Computers.computers_id

Clients ↔ Orders (1 ко многим)

Один клиент может делать много заказов

Один заказ принадлежит одному клиенту

Orders.client_id → Clients.clients_id

Sessions ↔ Orders (1 ко многим)

Одна сессия может иметь несколько заказов

Заказ может быть без сессии (для покупок без аренды)

Orders.session_id → Sessions.sessions_id

Orders ↔ Orders_Item (1 ко многим)

Один заказ может содержать несколько позиций

Одна позиция принадлежит одному заказу

Orders_Item.orders_id → Orders.orders_id

Services ↔ Orders_Item (1 ко многим)

Одна услуга может быть в разных позициях заказов

Одна позиция ссылается на одну услугу

Orders_Item.service_id → Services.services_id
